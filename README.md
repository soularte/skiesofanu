## Запуск сайта локально

```
npm run dev
```

Открыть в браузере: http://localhost:4321/

Если порт занят — Астро сам возьмёт следующий (4322 и т.д.), об этом напишет в терминале.

Собрать финальную версию (проверка перед публикацией):

```
npm run build
```

## Структура контентных файлов

```
src/
  content/books/     ← файлы книг (.md)
  data/
    book-order.json  ← порядок книг в галерее
    links.md         ← соцсети, литпорталы, контакты
    news.md          ← новостной баннер на главной
    telegram.md      ← посты телеграм-канала
public/
  images/            ← все картинки (обложки, фото авторов, персонажи)
    authors/         ← фото авторов
    characters-*/    ← фото персонажей (отдельная папка на каждый цикл)
```

## Как добавить новую книгу (отдельную, не цикл)

### Шаг 1. Добавить обложку

Скопировать файл обложки в `public/images/`. Назвать понятно, например `08 Новая книга.jpg`.

### Шаг 2. Создать файл книги

Создать файл `src/content/books/novayakniga.md` (имя файла — латиницей, без пробелов, это будет URL: `/books/novayakniga`).

Содержимое файла:

```yaml
---
title: "Название книги"
cover: "08 Новая книга.jpg"
genres: ["Жанр 1", "Жанр 2"]
logline: "Одна строка — суть книги."
shortDescription: "Короткая аннотация — 1-2 предложения. Показывается в карточке."
marketplace: true
links:
  - label: "ЛитРес"
    url: "https://litres.ru/..."
  - label: "Автор.Тудей"
    url: "https://author.today/..."
---

Полная аннотация. Показывается на странице книги.
```

> `marketplace: true` — добавляет бейдж «Маркетплейсы» над ссылками. Если книга не на маркетплейсах, поставь `false` или удели строку.
> Жанров максимум 3.

### Шаг 3. Добавить в порядок галереи

Сайт не показывает книги автоматически в порядке добавления — порядок задаётся вручную через `src/data/book-order.json`. Если не добавить slug сюда, книга появится в конце галереи.

Открыть файл и добавить slug (имя файла без `.md`) в нужное место:

```json
[
  "aviatory",
  "virdolog",
  "novayakniga"
]
```

## Как добавить книгу в существующий цикл

Открыть `src/content/books/aviatory.md` (или нужный цикл). Найти раздел `books:` и добавить новую запись по образцу:

```yaml
books:
  - title: "Авиаторы Его Величества. Книга первая"
    cover: "01 Авиаторы Его Величества.jpg"
    marketplace: true
    genres: ["Стимпанк", "Приключения"]   # опц., макс. 3
    logline: "Логлайн книги."
    shortDescription: "Короткая аннотация."
    description: "Полная аннотация — показывается на странице цикла."
    links:
      - label: "ЛитРес"
        url: "https://litres.ru/..."

  - title: "Авиаторы Его Величества. Книга вторая"   ← добавить сюда
    cover: "новая обложка.jpg"
    marketplace: false
    logline: "..."
    shortDescription: "..."
    description: "..."
    links:
      - label: "ЛитРес"
        url: "#"
```

Также добавить книгу в `readingOrder:` в том же файле.

## Как создать новый цикл

> Если поле `series` задано, на странице книги автоматически появляется золотой центрированный заголовок с названием цикла над обложкой. У одиночных книг этого заголовка нет.

### Шаг 1. Добавить обложку цикла в `public/images/`

### Шаг 2. Создать `src/content/books/названиецикла.md`

```yaml
---
title: "Название цикла (или первой книги)"
cover: "обложка.jpg"
marketplace: true
genres: ["Жанр 1", "Жанр 2"]
logline: "Логлайн цикла."
shortDescription: "Короткое описание — для карточки на главной."
series: "Название цикла"
seriesDescription: "Описание всего цикла. Показывается на странице цикла."
seriesLinks:
  - label: "ЛитРес"
    url: "https://litres.ru/..."
readingMap:
  label: "Карта чтения"
  url: "https://..."
readingOrder:
  - title: "Книга первая"
    labels: ["Основной сюжет"]
  - title: "Книга вторая"
    labels: ["Основной сюжет"]
world: "Описание мира. Показывается в секции «Мир»."
worldMaps:
  - label: "Карта мира"
    url: "#"
charactersFolder: "characters-названиецикла"
characters:
  - name: "Имя"
    role: "Роль"
    description: "Описание."
    photo: "hero.jpg"
characterMaps:
  - label: "Дерево связей"
    url: "#"
books:
  - title: "Книга первая"
    cover: "обложка1.jpg"
    marketplace: true
    logline: "..."
    shortDescription: "..."
    description: "..."
    links:
      - label: "ЛитРес"
        url: "#"
---
```

### Шаг 3. Добавить slug в `book-order.json`

Сайт не показывает книги автоматически в порядке добавления — порядок задаётся вручную. Если не добавить slug сюда, новый цикл появится в конце галереи.

Открыть `src/data/book-order.json` и добавить slug нового цикла в нужное место списка:

```json
["aviatory", "virdolog", "novyitsikl"]
```

Slug — это имя `.md` файла без расширения.

## Как отредактировать описание книги

**Отдельная книга** (например, `virdolog.md`):

- `logline` — строка-крючок, курсивом под заголовком
- `shortDescription` — в карточке галереи
- `mediumDescription` — средняя аннотация на странице списка `/books` (если не задана, используется тело `.md`)
**Книга внутри цикла** (например, в `aviatory.md`):

Найти нужную запись в `books:` и отредактировать:
- `logline` — курсивная строка под заголовком книги
- `shortDescription` — серым, краткое
- `description` — полный текст аннотации

## Как добавить/изменить ссылки на литплатформы

**Для отдельной книги** — в поле `links:` файла книги.

**Для всего цикла** — в поле `seriesLinks:` файла цикла.

```yaml
links:
  - label: "ЛитРес"
    url: "https://litres.ru/book/..."
  - label: "Автор.Тудей"
    url: "https://author.today/work/..."
  - label: "ЛитНет"
    url: "https://litnet.com/..."
  - label: "ЛитГород"
    url: "https://litgorod.ru/..."
```

## Как добавить новость на главную

Открыть `src/data/news.md`:

```yaml
---
visible: true
text: "Текст новости — одно-два предложения."
links:
  - label: "Читать"
    url: "https://..."
  - label: "Ещё ссылка"
    url: "https://..."
---
```

- Чтобы скрыть баннер: `visible: false`
- Ссылок можно сколько угодно, или удалить блок `links:` если ссылки не нужны

## Как добавить персонажей

### Шаг 1. Добавить фотографии

Скопировать фото в папку `public/images/characters-НАЗВАНИЕ/` (например `public/images/characters-aviators/`).

Если папки нет — создать.

### Шаг 2. Добавить персонажей в файл книги/цикла

```yaml
charactersFolder: "characters-aviators"
characters:
  - name: "Иван Крылов"
    role: "Главный герой"
    description: "Лётчик-истребитель, скрывающий дар к магии. Циничен снаружи, ранен внутри."
    photo: "ivan.jpg"
  - name: "Анна Тихомирова"
    role: "Главная героиня"
    description: "Шифровальщик флота. Умнее всех в комнате, и все об этом знают."
    photo: "anna.jpg"
```

> `photo` — только имя файла, без пути. Путь к папке берётся из `charactersFolder`.
> Если у персонажа нет фото — просто не пиши строку `photo:`.

**Порядок карточек** определяется порядком записей в списке `characters:` — первый в списке отображается первым слева. Чтобы переставить персонажей, достаточно поменять блоки местами в файле.

## Как изменить порядок карточек в галерее

Открыть `src/data/book-order.json` и переставить slug-и в нужном порядке:

```json
["aviatory", "virdolog", "stirayushchee", "muzyka", "gorod", "danlin"]
```

Slug — это имя `.md` файла без расширения.

## Как обновить соцсети и литпорталы

Открыть `src/data/links.md`. Найти нужный раздел и заменить `url: "#"` на реальную ссылку.

- `socials` — иконки соцсетей на главной и странице «Об авторах»
- `litportals.ksenia` — иконки литпорталов Ксении
- `litportals.vasily` — иконки литпорталов Василия

Примеры ссылок:

```yaml
socials:
  - label: "ВКонтакте"
    url: "https://vk.com/id..."
    icon: "vk"
  - label: "Telegram"
    url: "https://t.me/названиеканала"
    icon: "telegram"

litportals:
  ksenia:
    - label: "ЛитРес"
      url: "https://www.litres.ru/author/..."
    - label: "Автор.Тудей"
      url: "https://author.today/u/..."
  vasily:
    - label: "ЛитРес"
      url: "https://www.litres.ru/author/..."
```

> Иконки поддерживают ключи: `vk`, `telegram`, `threads`, `pikabu`, `dzen`. Значение `label` также отображается как всплывающая подсказка при наведении на иконку. Значение `label` для литпорталов должно совпадать с одним из: `ЛитРес`, `Автор.Тудей`, `ЛитНет`, `ЛитГород`.

Поля `contact.vk` и `contact.telegram` используются в строке «Связаться с нами» на главной и на странице об авторах.

## Как обновить Telegram-блок на главной

Открыть `src/data/telegram.md` и отредактировать нужные поля.

Пример:

```yaml
---
visible: true
title: "Наш Telegram-канал"
description: "Анонсы, отрывки, закулисье."
url: "https://t.me/названиеканала"
buttonLabel: "Подписаться"
---
```

- `visible: false` — скрыть блок без удаления данных

## Как добавить буктрейлер к книге или циклу

### Шаг 1. Положить видеофайл

Скопировать видео в папку `public/videos/`. Если папки нет — создать.

Например: `public/videos/aviatory-trailer.mp4`

### Шаг 2. Добавить поле в файл книги/цикла

```yaml
booktrailer:
  file: "aviatory-trailer.mp4"
  orientation: "horizontal"   # или "vertical" для вертикального видео
```

Секция появится на странице книги сразу после блока «О цикле».

Чтобы **временно скрыть** секцию, не удаляя данные:

```yaml
booktrailer:
  visible: false
  file: "aviatory-trailer.mp4"
  orientation: "horizontal"
```

> Если поля `booktrailer:` нет совсем — секция не показывается.
