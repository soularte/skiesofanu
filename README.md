## ⚠️ Перед первым запуском

Прежде чем выкладывать сайт в интернет, замени заглушку домена в **двух** файлах.

### 1. `astro.config.mjs`

Найди строку:
```js
site: 'https://example.com',
```
Замени `example.com` на реальный домен, например:
```js
site: 'https://bortzhurnal.ru',
```

### 2. `public/robots.txt`

Найди строку:
```
Sitemap: https://example.com/sitemap-index.xml
```
Замени так же:
```
Sitemap: https://bortzhurnal.ru/sitemap-index.xml
```

> Если домен ещё не выбран — оставь `example.com`. Сайт будет работать, просто карта сайта для поисковиков будет указывать неправильно.

## Запуск сайта локально

Открой терминал в папке проекта и запусти:

```
npm run dev
```

Открой в браузере: **http://localhost:4321/**

Если порт занят — Астро возьмёт следующий (4322 и т.д.) и напишет в терминале.

Проверить финальную сборку перед публикацией:

```
npm run build
```

## Карта файлов — что где менять

```
src/
  content/books/        ← файлы книг и циклов (.md)
  data/
    site.md             ← название сайта, аналитика, плашка конфиденциальности
    authors.md          ← имена, фото, биография, влияния авторов
    links.md            ← соцсети, литпорталы, ссылки «Связаться с нами»
    news.md             ← новостная плашка на главной странице
    telegram.md         ← посты Telegram в блоке «Последние новости»
public/
  images/               ← все картинки
    authors/            ← фото авторов (kk.jpg, vz.jpg, favis.jpg, favi.jpg)
    characters/         ← фото персонажей цикла «Небеса Ану»
  videos/               ← видео-буктрейлеры (.mp4)
  robots.txt            ← инструкция для поисковых роботов
astro.config.mjs        ← домен сайта (менять один раз при запуске)
```

## site.md — настройки сайта

Файл: `src/data/site.md`

```yaml
---
# Название сайта — в шапке и в заголовке вкладки браузера
siteName: "Бортжурнал"

# Подзаголовок под названием в шапке (убери строку, если не нужен)
siteTagline: "Ксения Котова · Василий Зеленков"

# Описание по умолчанию — для страниц без собственного описания (SEO)
defaultDescription: "Официальный сайт авторского дуэта..."

# ID счётчика Яндекс.Метрики. Оставь пустым "", чтобы отключить аналитику
metricsId: ""

# Текст плашки конфиденциальности внизу страницы
privacyText: "Этот сайт использует Яндекс.Метрику..."
---
```

**Как включить Яндекс.Метрику:** получи ID счётчика в кабинете Метрики и впиши его:
```yaml
metricsId: "12345678"
```
Плашка и счётчик включатся автоматически.

  name: "Василий Зеленков"

  # bioUrl: "https://..."
  # bioLabel: "Биография"

- мета-тег `author` (SEO)
- JSON-LD разметку страниц книг

## links.md — соцсети и литпорталы

Файл: `src/data/links.md`

```yaml
---
# Иконки соцсетей на главной и странице «Об авторах»
# Доступные ключи icon: vk | telegram | threads | pikabu | dzen
socials:
  - label: "ВКонтакте"
    url: "https://vk.com/skiesofanu"
    icon: "vk"
  - label: "Телеграм"
    url: "https://t.me/skiesofanu"
    icon: "telegram"

# Иконки литпорталов — значение label должно быть точно одним из:
# ЛитРес | Автор.Тудей | ЛитНет | ЛитГород
litportals:
  ksenia:
    - label: "ЛитРес"
      url: "https://www.litres.ru/author/..."
    - label: "Автор.Тудей"
      url: "https://author.today/u/..."
  vasily:
    - label: "ЛитРес"
      url: "https://www.litres.ru/author/..."

# Ссылки в строке «Связаться с нами» на главной и странице об авторах
contact:
  vk: "https://vk.com/..."
  telegram: "https://t.me/..."
---
```

> **Важно:** поле `label` в литпорталах — это не просто текст, это ключ для иконки.
> Если написать что-то отличное от списка выше (например, «Литрес» вместо «ЛитРес») — иконка не появится.

> Чтобы убрать иконку соцсети — просто удали соответствующий блок из списка `socials:`.

## news.md — новостная плашка

Файл: `src/data/news.md`

Плашка отображается вверху главной страницы под шапкой.

```yaml
---
# Поставь false, чтобы скрыть плашку (или true, чтобы показать)
visible: true

# Текст новости
text: "Вышла новая книга! Уже доступна на ЛитРес."

# Кнопки-ссылки рядом с текстом (удали весь блок links, если кнопки не нужны)
links:
  - label: "Читать на ЛитРес"
    url: "https://litres.ru/..."
---
```

Чтобы скрыть плашку без удаления текста — поставь `visible: false`.

## telegram.md — посты в блоке «Последние новости»

Файл: `src/data/telegram.md`

Блок на главной показывает три виджета Telegram-постов. `postId` — это часть ссылки на пост: `t.me/КАНАЛ/НОМЕР`.

```yaml
---
posts:
  digest:
    label: "Последний дайджест"
    postId: "skiesofanu/4498"   # ← ссылка: t.me/skiesofanu/4498
  current:
    label: "Актуальное"
    postId: "skiesofanu/5102"
  intro:
    label: "Знакомство с каналом"
    postId: "skiesofanu/1"
---
```

Чтобы поменять пост — обнови `postId` на номер нужного поста из канала.

## Книги: как добавить отдельную книгу

### Шаг 1. Добавить обложку

Скопируй файл обложки в `public/images/`. Назови понятно, например `08 Новая книга.jpg`.

### Шаг 2. Создать файл книги

Создай файл в `src/content/books/` — например `novayakniga.md`.
Имя файла латиницей, без пробелов — оно станет URL страницы: `/books/novayakniga`.

```yaml
---
title: "Название книги"
cover: "08 Новая книга.jpg"    # имя файла из public/images/
displayOrder: 7                # порядковый номер в галерее (чем меньше — тем левее)
genres: ["Жанр 1", "Жанр 2"]  # максимум 3 жанра
logline: "Одна строка — суть книги."
shortDescription: "Короткая аннотация — 1-2 предложения. Показывается в карточке."
marketplace: true              # true = бейдж «Маркетплейсы» над ссылками, false = без бейджа
links:
  - label: "ЛитРес"
    url: "https://litres.ru/..."
  - label: "Автор.Тудей"
    url: "https://author.today/..."
---

Полная аннотация. Показывается на странице книги. Можно писать несколько абзацев.
```

### Текущий порядок `displayOrder` в галерее

| Книга | displayOrder |
|---|---|
| Авиаторы Его Величества | 1 |
| Вирдолог | 2 |
| Стирающее поветрие | 3 |
| Пусть эта музыка стихнет | 4 |
| Город без имени | 5 |
| Данлин | 6 |

Новая книга = следующий номер (7, 8 и т.д.), или вставь в середину и перенумеруй нужные.

## Книги: как добавить книгу в существующий цикл

Открой `src/content/books/aviatory.md` (или нужный цикл). Найди раздел `books:` и добавь новый блок в конец списка:

```yaml
books:
  # ... существующие книги ...

  # ← добавь новую книгу сюда
  - title: "Авиаторы Его Величества: Новая часть"
    cover: "новаяобложка.jpg"
    marketplace: false
    logline: "Логлайн книги."
    shortDescription: "Короткая аннотация."
    description: "Полная аннотация — показывается на странице цикла."
    links:
      - label: "ЛитРес"
        url: "https://litres.ru/..."
```

Также добавь книгу в `readingOrder:` в том же файле.

## Книги: порядок в галерее

Порядок карточек задаётся полем `displayOrder` в каждом `.md`-файле. Чем меньше число — тем левее карточка.

Чтобы поменять порядок — открой нужные файлы и измени числа:

```yaml
# virdolog.md — хочу поставить первым
displayOrder: 1

# aviatory.md — сдвигаю на второе место
displayOrder: 2
```

## Персонажи: добавить или изменить

### Шаг 1. Добавить фотографии

Скопируй фото в папку `public/images/characters/` (для «Небес Ану»).
Для нового цикла создай отдельную папку и укажи её в поле `charactersFolder:`.

### Шаг 2. Добавить персонажа в файл книги

```yaml
charactersFolder: "characters"
characters:
  - name: "Лем Декс"
    role: "Вольный капитан"
    description: "Умна, опасна, остра на язык."
    photo: "lem.jpg"    # только имя файла, папка берётся из charactersFolder

  - name: "Новый персонаж"
    role: "Роль"
    description: "Описание."
    # photo не указана — карточка покажет имя на тёмном фоне
```

**Порядок карточек** = порядок записей в списке. Поменяй блоки местами, чтобы переставить.

## Карты: добавить ссылки на карты мира и дерево связей

Кнопки под секциями «Мир» и «Персонажи» открывают ссылку **в новой вкладке браузера**.

Проще всего — положить картинку карты прямо в `public/images/` и указать путь к ней. Браузер откроет изображение на весь экран, читатель сможет приблизить его.

```yaml
worldMaps:
  - label: "Карта мира"
    url: "/images/world-map.jpg"   # файл лежит в public/images/world-map.jpg

characterMaps:
  - label: "Дерево связей"
    url: "/images/characters-tree.jpg"

readingMap:
  label: "Карта чтения"
  url: "/images/reading-map.jpg"
```

Можно указать внешнюю ссылку — например, на Google Диск или любой другой сервис:

```yaml
worldMaps:
  - label: "Карта мира"
    url: "https://drive.google.com/..."
```

Несколько кнопок сразу:

```yaml
worldMaps:
  - label: "Карта мира"
    url: "/images/world-map.jpg"
  - label: "Карта городов"
    url: "/images/cities-map.jpg"
```

## Буктрейлер: добавить видео

### Шаг 1. Положить видеофайл в `public/videos/`

Создай папку `public/videos/`, если её нет. Скопируй туда `.mp4`-файл.

### Шаг 2. Добавить поле в файл книги/цикла

```yaml
booktrailer:
  file: "aviatory-trailer.mp4"
  orientation: "horizontal"   # горизонтальное видео
  # orientation: "vertical"   # вертикальное (Reels/TikTok-формат)
```

Секция «Буктрейлер» появится на странице книги автоматически.

Чтобы **временно скрыть** без удаления данных:
```yaml
booktrailer:
  visible: false
  file: "aviatory-trailer.mp4"
  orientation: "horizontal"
```

## Отзывы: добавить цитаты читателей

В файл книги или цикла добавь поле `reviews:`:

```yaml
reviews:
  - text: "Не могла оторваться! Мир проработан до мелочей, персонажи живые."
    author: "Читательница с ЛитРес"
  - text: "Редкое сочетание приключений и глубокой психологии."
    author: "Рецензент книжного клуба"
  - text: "Лучшее, что я читала в жанре за последние годы."
    author: "Подписчица Telegram-канала"
```

Отзывы показываются в три колонки в секции «Отзывы» на странице книги.
Если отзывов нет вообще — удали поле `reviews:`, секция автоматически скроется.

## Фото авторов: заменить

Замени файлы в папке `public/images/authors/`:

| Файл | Что это |
|---|---|
| `kk.jpg` | Фото Ксении Котовой (страница «Об авторах» и hero-блок) |
| `vz.jpg` | Фото Василия Зеленкова (страница «Об авторах» и hero-блок) |
